# Тренажёры — учёт прохождений

Веб-приложение (работает в браузере, без установки и без интернета после первого открытия).
Хранит пациентов и три теста: «Слова», «Мелкая моторика», «Рисовалка».
Все данные лежат локально в браузере (IndexedDB). Экспорт — CSV и ZIP.

## Файлы
- `index.html` — само приложение (можно запускать отдельно)
- `manifest.webmanifest`, `sw.js`, `icon-*.png` — для установки как приложение и офлайн-режима

## Быстрый запуск на ПК (для проверки)
В этой папке выполните:

    python -m http.server 8000

Откройте в браузере: http://localhost:8000
(микрофон для теста «Слова» работает на localhost — это «защищённый контекст»).

## Запуск на планшете/телефоне
Микрофон, стилус и установка как приложение требуют HTTPS **или** localhost.
Самый простой способ — выложить папку на бесплатный HTTPS-хостинг (GitHub Pages),
либо запустить локальный сервер прямо на планшете (Termux). Подробности — в ответе ассистента.

## Стилус (планшет Huawei MatePad Air, DBY2-W09)
Нужен активный стилус Huawei M-Pencil (3-го поколения; 2-е тоже подходит).
Обычная ёмкостная «палка» не даёт отличить перо от ладони — отключения ладони не будет.

## Синхронизация с узлом

Каждый завершённый тест сам уезжает на узел хранилища. Сети нет — копится в очереди и
доезжает позже. Локальная копия и экспорт CSV/ZIP работают как раньше: это резервный канал,
данные из браузера после отправки **не удаляются**.

Пока узел не настроен, приложение ведёт себя ровно как прежде: никаких запросов, никакой
очереди, никаких значков.

### Подключение

1. В админке узла: «Ключи» → «+ Планшет» → на экране появится QR.
2. В приложении: «Настройки» → «Сканировать QR». Ручной ввод адреса и ключа — запасной путь.
3. «Проверить связь». Возможные ответы:
   - «Связь с узлом есть» (в т.ч. при HTTP 401 — это ожидаемо, эндпоинт статуса требует входа
     оператора; ключ проверится при первой отправке теста);
   - «не установлен сертификат» — откройте `https://<адрес узла>/setup/cert` и установите его;
   - «узел не разрешает запросы с этого адреса (CORS)» — на **узле** нужно добавить origin
     планшета в `PLANSHET_CORS_ORIGINS` (см. ниже).

Ключ хранится в IndexedDB планшета и уходит только в заголовке `X-Api-Key`. `device_id`
(вида `pl-1a2b3c4d`) генерируется один раз при первом запуске.

**CORS на узле.** Приложение планшета открыто со своего адреса (например `http://localhost:8000`),
а ходит на узел — это кросс-доменные запросы. Узел должен разрешить origin планшета, иначе браузер
блокирует и отправку. На стороне `storage-core` в `.env`:

    PLANSHET_CORS_ORIGINS=http://localhost:8000     # origin, с которого открыт САМ планшет (не адрес узла)

затем `docker compose up -d webapp`. Несколько адресов — через запятую.

### Как это выглядит в работе

- Пациента заводит оператор вручную (ФИО/код на главном экране) — как и раньше. Справочник
  пациентов с узла на планшет не выгружается; сшивку записи с человеком делает узел по ФИО.
- На карточке пациента (когда узел настроен) — «Точка» (1/2) и «Нагрузка» (до/после). Значения
  запоминаются до смены пациента и уходят вместе с каждым тестом.
- В шапке «Не отправлено: N» — сколько тестов ждёт отправки. Нажатие открывает очередь.
- Отправка идёт при запуске, раз в 60 секунд и при появлении сети. Ошибка ключа (401/403)
  останавливает отправку и показывает баннер: исправьте ключ в настройках, очередь пойдёт сама.
- Тесты, сделанные **до** подключения узла, в очередь сами не встают. Кнопка «Поставить в
  очередь все локальные тесты» в настройках отправит их разом.

### Проверка без узла

    python -m http.server 8000                     # приложение
    python tests/mock_node.py --port 8765 --key devkey --dir tests/_batches

В настройках приложения укажите `http://localhost:8765` и ключ `devkey`.

Заглушка проверяет батч теми же правилами, что и настоящий узел (формат `batch_id`, совпадение
частей с `manifest.files`, sha256 и размер каждой части) и раскладывает принятое в
`tests/_batches/<device_id>/planshet/<batch_id>/`. Режимы сбоев:

    curl -X POST localhost:8765/_control -d '{"fail_next": 2}'   # следующие 2 отправки -> 500
    curl -X POST localhost:8765/_control -d '{"unauth": true}'   # всё -> 401

Чек-лист (см. `docs/SYNC_SPEC.md` §6):

1. **Онлайн.** Пройдите тест → в `tests/_batches/` появился батч с `manifest.json` и `_complete`.
2. **Офлайн.** Остановите заглушку, пройдите три теста → бейдж «Не отправлено: 3». Запустите
   заглушку → все три доехали, порядок `batch_id` совпал с порядком прохождения.
3. **Повтор.** Отправьте тот же тест ещё раз (в настройках «Повторить») → узел отвечает
   `duplicate`, запись помечается отправленной, второго батча на диске не появляется.
4. **Плохой ключ.** `--unauth` → статус «ключ отклонён», баннер в шапке, отправка не долбит узел.

Автотесты сборки батча, апгрейда БД и QR-декодера — `tests/test.html`: откройте
`http://localhost:8000/tests/test.html` (нужен сервер: страница читает `../index.html`).
